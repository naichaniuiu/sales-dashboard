## 踩坑经验

- **欠款数据来源**：必须用"欠款数据"表汇总，不能用"26财年Q1业绩"表中的欠款金额列。
  两者差异巨大（如西安站欠款：业绩表79.92万 vs 欠款表339.61万，相差259万）。
  后续所有涉及欠款金额、账龄（30天内/30-90天/90-180天/180天以上）的计算，
  均需从"欠款数据"表按销售员维度汇总。

- **generate_sales_detail.py**：欠款金额字段 `debt` 已改为取账龄汇总 `total_debt`，不再从业绩表欠款列读取。
- **欠款数据正确口径**：欠款表全部人员（在职+离职），保留负数欠款（超额回款），跨部门调动人员以欠款表部门归属为准（如刘美希、危昶在欠款表属成都站但在业绩表属重庆站）
- **sales_detail.json 嵌入 HTML**：为避免 fetch/file:// 跨域问题，销售员明细数据已直接嵌入 HTML 页面作为 JS 变量，不再依赖外部 JSON 文件
- **部门行点击行为**：业绩对比表部门行现已改为 `showSalesDetail()` 直接弹出销售员明细弹窗，而非 `showDeptDetail()` 部门概览弹窗
- **弹窗层级问题（重要）**：`salesModal` 销售员明细弹窗不能嵌套在 `modalOverlay` 部门概览弹窗内部。因为 `modalOverlay` 默认 `display:none`，子元素即使 `display:flex` 也会被父容器隐藏。必须将 `salesModal` 作为独立顶层元素放在 `modalOverlay` 之外。

- **三模块独立下钻视图**：`showSalesDetail(dept, type)` 根据 Tab 类型分发到不同渲染函数：
  - `'perf'` → `renderSalesPerf`：业绩/回款/单数/状态，按业绩排序
  - `'debt'` → `renderSalesDebt`：合计欠款/30天内/30-90天/90-180天/180天以上/风险状态，按欠款排序
  - `'cycle'` → `renderSalesCycle`：回款金额/欠款金额/回款周期/状态（回款周期=欠款×90/回款），按周期排序
  - 各 Tab 的部门行 onclick 均传入对应类型参数；部门概览弹窗按钮使用 `getCurrentTab()` 自动判断当前 Tab
  - **函数命名不能冲突**：JavaScript 不支持函数重载，同名函数后者覆盖前者。销售员明细渲染函数必须用 `renderSalesPerf`/`renderSalesDebt`/`renderSalesCycle`，不能与原有的 `renderDebtTable()`、`renderCycleTable()`（渲染分析表格用）同名

- **超90天部门数KPI卡片点击下钻**：回款周期分析模块的"超90天部门数"卡片已添加 `onclick="showOver90Depts()"`，点击后弹出回款周期>90天的部门列表弹窗，列表中部门行支持进一步点击下钻到销售员回款周期明细
